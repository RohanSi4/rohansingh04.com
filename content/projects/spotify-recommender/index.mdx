The best recommendations usually start with one song I cannot stop playing.
Signal takes that song and gives me a few honest paths outward instead of
another generic playlist.

The [live app](https://signal-recommender.vercel.app) searches Spotify's real
catalog. Pick any song and Signal pulls from that artist's releases, follows
the collaborators it finds there, and brings back tracks with real artwork,
album details, release dates, and direct Spotify links. Public search works
without an account.

![Signal building a real mix from Spotify releases and artist connections](/projects/spotify-signal.png)

## how the mix comes together

Signal starts with the artists on the selected track and walks through their
albums and singles. It looks for recurring collaborators, follows a few of
those connections into their catalogs, removes duplicates and low-value
versions, and keeps one album or artist from taking over the whole result.

Every card says why it appeared, such as more from the seed artist or connected
through a featured artist. Those are real catalog relationships. The interface
does not show a made-up similarity percentage or pretend Spotify exposes an
audio match that the current API cannot provide.

## a personal path for invited listeners

Spotify login is optional. For an invited account, Signal reads the listener's
medium-term top tracks, finds the artists that show up most, and looks past the
obvious favorites for songs that are not already in that top rotation. The
result is a deeper-cuts mix grounded in what that person actually plays.

Spotify currently limits new development apps to five invited listeners, so
that personal path is a small beta. The public song search and song-based mix
still work for everyone.

## building around the API that really exists

The current product is a Next.js and TypeScript app. Spotify credentials stay
on the server behind route handlers, account sessions use encrypted cookies,
OAuth state is checked on callback, expired access tokens can refresh, and
public endpoints have basic rate limits. The browser only receives the track
details it needs to render the experience.

Signal began as a Python audio-feature ranking prototype. Spotify later removed
access to several recommendation and analysis endpoints for most new apps, so I
rebuilt the live product around catalog search, releases, top tracks, and real
artist connections. The original Python work is still in the repository, but
the Spotify-backed web app is now the project I want people to use.
