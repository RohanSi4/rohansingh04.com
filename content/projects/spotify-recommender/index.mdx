I built this project to learn how recommendations can emerge from listening
behavior rather than a hand-written list of rules. The system uses Python and
collaborative filtering to work from listening history and find relationships
between users and music.

The scope was intentionally smaller than my later recommendation work: this was
about understanding the core ML idea, not building a production serving stack.
It gave me a practical starting point for thinking about sparse behavioral data,
similarity, and the difference between a model that runs and a recommendation
someone might actually want.

The project is complete and the source remains available on GitHub. I would
approach the evaluation and product layer differently today, but that is also why
I keep it here: it shows the point where recommendation systems became more than
a classroom concept for me.
