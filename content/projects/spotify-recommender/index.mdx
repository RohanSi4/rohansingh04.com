The music I want changes with the moment. Sometimes I want more songs like one
I already love. Sometimes I know the energy I want but not the song. I built
Signal so either starting point could lead somewhere useful, and so the result
would explain itself instead of feeling random.

Pick a reference track or a mood and the app builds a target across nine audio
features. It normalizes the different scales, applies documented weights, ranks
the candidate catalog, and shows the two closest traits for every result.

The [Signal browser demo](https://signal-recommender.vercel.app) ports that same
weighted nine-feature method to TypeScript. Choose any of 72 deterministic demo
tracks or start from a mood, adjust the blend, and rank the catalog entirely in
your browser. The interaction is real, nothing is sent to a server, and no
Spotify login is required.

## making it work without a perfect API setup

Spotify changed access to several of the endpoints this project originally
used. New development apps often cannot read Audio Features or use Spotify's
Recommendations endpoint at all. Instead of hiding that limitation, I split
the ranking engine from the Spotify connection.

The default app now runs against a deterministic 72-track demo catalog. It
needs no account, credentials, or network connection. Spotify OAuth is still
there for library access and private playlist export when an app has the right
permissions, but the actual recommendation work can be opened, tested, and
understood by anyone.

The Python engine remains the reference implementation for Spotify experiments,
the evaluator, and the test suite. The hosted TypeScript experience is the
public proof layer. It uses made-up track names and feature profiles because
Spotify's development rules do not allow republishing its catalog as an open
portfolio dataset.

## what changed in the rebuild

- fixed a ranking bug that excluded the entire user library instead of only the
  selected seed tracks
- moved the recommendation logic into a credential-free core
- added an offline evaluator and a fixed synthetic regression set
- made every result include its score and plain-language match reasons
- replaced the older UI with one focused Streamlit flow
- added nine tests around ranking order, exclusions, invalid data, pagination,
  API restrictions, and evaluator determinism

The included fixture reaches 100% hit-rate@5 and 0.993 mean reciprocal rank,
but those numbers are only regression checks on deliberately separated
synthetic clusters. They are not a claim about real listeners. The result I
care about is simpler: the project now works out of the box, and its claims can
be reproduced from the repository.
